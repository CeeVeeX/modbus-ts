在 Modbus 里，“逻辑地址”和“物理地址”经常被混用，但实际上是两个层面的概念。

你这个报文里：

```text
01 9C 41 00 01
```

真正发送的是：

```text
9C41
```

这属于 **物理地址（Protocol Address）**。

而很多文档写成：

```text
40001
00001
30001
10001
```

这属于 **逻辑地址（Reference Address）**。

下面系统说明。

---

# 一、Modbus 的四类逻辑地址

Modbus 规范定义了四种数据区：

| 区域               | 前缀    | 功能    | 功能码      |
| ---------------- | ----- | ----- | -------- |
| Coil             | 0xxxx | 读写线圈  | 01/05/15 |
| Discrete Input   | 1xxxx | 只读输入  | 02       |
| Input Register   | 3xxxx | 输入寄存器 | 04       |
| Holding Register | 4xxxx | 保持寄存器 | 03/06/16 |

例如：

| 逻辑地址  | 含义       |
| ----- | -------- |
| 00001 | 第1个 Coil |
| 10001 | 第1个离散输入  |
| 30001 | 第1个输入寄存器 |
| 40001 | 第1个保持寄存器 |

这些地址：

* 主要给人看
* 属于“逻辑地址”
* 报文里并不直接发送

---

# 二、报文里发送的是“物理地址”

Modbus PDU 中：

```text
功能码 + 起始地址 + 数量
```

这里的地址是：

# 0-based Offset（偏移地址）

即：

```text
实际发送地址 = 逻辑地址去掉前缀后再减1
```

---

# 三、地址转换规则

## 1）Holding Register（4xxxx）

例如：

| 逻辑地址  | 物理地址   |
| ----- | ------ |
| 40001 | 0x0000 |
| 40002 | 0x0001 |
| 40100 | 0x0063 |

公式：

```text
物理地址 = 逻辑地址 - 40001
```

---

## 2）Coil（0xxxx）

| 逻辑地址  | 物理地址   |
| ----- | ------ |
| 00001 | 0x0000 |
| 00002 | 0x0001 |

公式：

```text
物理地址 = 逻辑地址 - 1
```

---

# 四、结合你的报文分析

你的请求：

```text
01 9C 41 00 01
```

拆解：

| 字段   | 值      |
| ---- | ------ |
| 功能码  | 0x01   |
| 起始地址 | 0x9C41 |
| 数量   | 1      |

---

## 物理地址

```text
0x9C41 = 40001（十进制）
```

这是：

# 物理偏移地址 = 40001

---

## 对应逻辑地址？

因为功能码是：

```text
0x01 → Coil
```

所以逻辑地址应该是：

```text
40001 + 1
= 40002
```

即：

# 逻辑 Coil 地址 = 40002

注意：

这里不是 Holding Register 的 40002。

因为：

* 功能码决定数据区
* 地址本身只是 offset

---

# 五、这是 Modbus 最容易混乱的地方

很多人会误以为：

```text
9C41 → 40001 → Holding Register 40001
```

其实错了。

因为：

# “40001”这个前缀只在人类文档中有意义

真正决定区域的是：

| 功能码 | 数据区              |
| --- | ---------------- |
| 01  | Coil             |
| 02  | Discrete Input   |
| 03  | Holding Register |
| 04  | Input Register   |

---

# 六、你的报文真实含义

请求：

```text
01 9C41 0001
```

含义：

```text
读取 Coil 区
offset = 0x9C41
数量 = 1
```

如果按传统逻辑地址写法：

```text
读取 Coil 40002
```

---

# 七、为什么会有两套地址体系

历史原因。

早期 Modbus 文档为了区分数据区：

* 0xxxx → Coil
* 1xxxx → Input
* 3xxxx → Input Register
* 4xxxx → Holding Register

但协议报文里：

* 不发送前缀
* 只发送 offset

因此形成：

# “逻辑地址” vs “物理地址”

---

# 八、工程里最重要的经验

很多 PLC/HMI/驱动：

* 有的要求填逻辑地址
* 有的要求填物理地址

例如：

| 软件        | 地址填写   |
| --------- | ------ |
| Modscan   | 常用逻辑地址 |
| libmodbus | 用物理地址  |
| PLC驱动     | 厂家各不相同 |

所以必须确认：

# “地址是否从0开始”

这是 Modbus 调试最大坑之一。

---

# 九、最实用判断法

看报文：

```text
03 00 00 00 01
```

如果你认为这是：

* 40001 → 对
* 因为功能码03决定是Holding Register

而不是：

* 00001

---

# 十、一句话总结

## 逻辑地址

给人看的：

```text
40001 / 30001 / 00001
```

## 物理地址

报文真正发送：

```text
0000 / 0001 / 0063
```

## 功能码决定数据区

地址本身不决定数据类型。
